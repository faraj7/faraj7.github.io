Para quem não está familiarizado com o conceito de breakpoint, uma breve descrição é a seguinte:

Breakpoint é um ponto do código definido para que a execução do mesmo pare imediatamente antes da instrução selecionada.

Assim, o trabalho de depuração e correção de erros é facilitado, pois detectar o trecho de código defeituoso é realizado de maneira mais organizada.

Para que possamos estabelecer e depurar o código utilizando o CodeWarrior é possível seguir os seguintes passos:

1. Selecionar a linha de código para criar um breakpoint e clicar com o botão direito na barra lateral esquerda:

2. Clicar em Add Breakpoint:

3. Clicar em OK:

4. Agora com o breakpoint definido podemos executar o código, porém devemos realizar essa tarefa em modo de depuração, clicando no ícone do "bug":

5. Feito isso, o CodeWarrior troca de interface e apresenta os recursos do ambiente de depuração:

6. O programa, nesse ambiente, não começa em execução, e para que ele execute até parar no primeiro breakpoint definido devemos clicar no botão:

7. Agora o CodeWarrior executou nosso código até encontrar o primeiro breakpoint e deixou a linha em destaque:

8. Nesse momento é possível avaliar valores de variáveis e outros atributos relacionados a execução do código na seguinte seção do ambiente do CodeWarrior. No caso do programa que acende os 8 LEDs vermelhos, podemos observar o valor do registrador conforme a execução (O conteúdo do registrador é armazenado na variável s). Para alterar a visualização para binário, basta clicar com o botão direito sobre a variável e alterar o "Format" para "Binary":

9. Além de monitorar as variáveis, também é possível monitorar uma posição de memória, já que é onde escrevemos quando vamos alterar o valor de um registrador para apagar ou acender LEDs. Para isso, acesse a aba "Memory" e clique no ícone +:

10. Agora, devemos inserir o endereço de memória a ser monitorado. Nesse caso, vamos monitorar o GPIOC_PDOR para verificar os bits da Porta C durante o loop. Esse registrador tem como endereço 0x400FF080:

11. A aba "Memory" passa a exibir os valores armazenados na memória:

12. Conforme o código é executado, o conteúdo do endereço de memória se altera e é indicado pelo monitor de memória criado:

13. Então, é possível decidir se a execução deve continuar normalmente até o próximo breakpoint (ou final do código), ou de algum outro modo, como a execução passo a passo, utilizando os botões.

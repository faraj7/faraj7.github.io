O objetivo deste tutorial é fornecer as noções básicas para que o aluno possa criar um novo projeto, fazer a compilação cruzada do programa main.c e executá-lo na placa. São 8 passos:

1. Criar um novo projeto

2. Dar o nome ao projeto

3. Selecionar o microcontrolador MKL25Z128

4. Selecionar a interface openSDA

5. Copiar o arquivo main.c do Piazza

6. Compilar o programa main.c

7. Ligar e carregar o programa no microcontrolador

8. Executar e testar o programa

A seguir são dados os detalhes de cada passo:

1. Criar um novo projeto do tipo Bareboard:

Selecione o menu File > New > Bareboard Project

2. Escolher um nome para o projeto:

Escolha um nome sugestivo e sem espaços ou pontuações como por exemplo: aula1, piscaled, etc.

3. Selecionar a placa que será o alvo do projeto.

Neste curso, será utilizada a FRDM-KL25Z com 128kb de FLASH: código MKL25Z128

Dica: uma forma rápida de procurar é preencher no campo de busca type filter text

4. Selecionar qual será a interface entre a placa e o CodeWarrior:

Deve ser selecionada a opção openSDA.

Para o propósito deste curso, o usuário pode finalizar a criação do projeto nesta etapa (não é necessário seguir para as próximas janelas):

Neste momento o CodeWarrior cria o seu projeto e apresenta a sua tela principal.

5. Copiar o conteúdo do main.c

Na janela CodeWarrior Project (marcado em preto na figura abaixo) irá aparecer o projeto criado e dentro dele tem uma pasta Sources contido um arquivo main.c já criado. Recorte o programa desejado (de acender LED vermelho) do Piazza e cole no arquivo main.c, apagando todo o seu conteúdo prévio.

6. Compilar o main.c

O programa main.c pode ser compilado tanto selecionando o botão do martelo (em azul na figura acima) ou selecionando o menu XXXXX.

7. Ligar e carregar no microcontrolador

O programa main precisa agora ser ligado com as demais bibliotecas e carregado via o openSDA para o microcontrolador. Isto é feito pelo botão assinalado em vermelho na figura acima.

8. Executar e testar o programa

A figura abaixo representa a interface de depuração, que é aberta automaticamente. Nela, você irá encontrar o código em linguagem de montagem (em vermelho), uma pequena janela que permite a visualização fácil das variáveis, registradores, breakpoints e conteúdo da memória (em preto). Os três botões destacados em azul são responsáveis por iniciar, pausar e terminar a execução do programa, respectivamente.

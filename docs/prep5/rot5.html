Objetivos:
  • Conhecer como funciona um controlador de display LCD
  • Analisar esquemático e relacioná-lo com o software necessário para a sua aplicação
  • Analisar o manual do LCD e desenvolver programa para sua interface
  • Projetar o programa de acordo com a temporização dos sinais especificada pelo fabricante do LCD
  • Consolidação dos conhecimento sobre a GPIO

  • Demonstração do prog4

  • Fazer o programa que inicializa o GPIO, inicializa o display LCD e escreve alguma mensagem

Programas a serem entregues na próxima aula:
  • Folder prog5: Programa que inicializa GPIO, LCD e escreve alguma mensagem
    • Atenção: o objetivo desta experiência é seguir as recomendações do fabricante do LCD e implementar as funções conforme especificado em relação às temporizações recomendadas. Não se deve colocar nem tempo a mais ou a menos.
  • Utilizar os seguintes nomes de funções:
    • void init_GPIO_LCD(); // inicializa as portas GPIO para operar com o LCD
    • void init_LCD(); // inicializa o LCD, 2 linhas, limpeza do display, etc.
      • ATENÇÃO: utilize a função sendcmd_LCD definida a seguir para implementar a init_LCD
    • void sendcmd_LCD( char c, int us); // envia código de comando para o LCD e espera us microssegundos
      • // Esta função deve colocar no início o sinal RS = 0, indicando que se estará enviando comandos para o LCD
    • void putchar_LCD( char c); // imprime o código ASCII guardado em c no LCD
      • // Esta função deve colocar no início o sinal RS = 1, indicando que se estará enviando dados ao LCD
  • Este programa deve utilizar as macros definidas pelo "derivative.h" que por sua vez inclue o "MKL25Z4.h" que contém as macros necessárias
    • Atenção estes dois arquivos não devem ser editados, pois eles fazem parte do sistema do CodeWarrior

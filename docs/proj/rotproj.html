A proposta de um projeto deve ser aprovada pela professor. Projeto proposto: Sistema de "chat" entre dois microcontroladores. Utilizar a segunda interface serial UART2 com seus sinais RX, TX e terra disponíveis na placa shield para interconectar dois microcontroladores. Utilizar a interface serial UART0 como terminal para teclar as mensagens, a visualização no terminal pode ser utilizada para mostrar alguma informação a critério do grupo. O display LCD deve ser utilizado para mostrar na primeira linha o que o colega está digitando e mostrar na segunda linha o que você está digitando. Utilizar o efeito de rolagem nestas linhas para que as mensagens sejam contínuas no display. Procurar fazer um esquema de sinalização de reconhecimento de transmissão e recepção de mensagem parecido com o do whatsapp. Se o cabo não estiver ligado ou se o outro microcontrolador não estiver funcionando, a mensagem não poderá ser transmitida. Quando o cabo for reconectado ou o microcontrolador for ligado, a mensagem deve então ser transmitida. As duas portas seriais devem funcionar por interrupção tanto de entrada (RX) como de saída (TX) assim como também a interrupção do Systick. Mostrar cronômetro junto e dar alguma função aos botões de pressão também é recomendável. Variações sobre este enunciado são permitidas assim como novas formas de interação entre os dois microcontroladores. Haverá uma primeira nota pelo que for entregue até a próxima aula e uma segunda nota até uma semana após a aula do projeto II. A nota do projeto será a média entre as duas notas. É possível entregar o projeto em qualquer momento antes destas datas. O que é esperado como entrega para a próxima aula (entregar no folder "proj1":
  • Uma descrição funcional do projeto proposto
  • Uma implementação bem simplificada de alguma parte crítica do projeto
Recomendações: Uma dos pontos principais que auxiliam no desenvolvimento de projetos é a sua estruturação, sua divisão por partes e o desenvolvimento de testes incrementais. Particularmente neste projeto que envolve a comunicação entre dois controladores, é muito difícil testar dois programas simultâneos, TX e RX, sendo que podem existir erros nos dois simultaneamente. A metodologia de desenvolvimento e testes do projeto deve ser escrita no relatório.
